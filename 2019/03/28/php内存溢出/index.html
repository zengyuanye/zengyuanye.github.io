memory_limit

memory_limit 限制php进程对于内存的使用

PHP 官网的解释

This sets the maximum amount of memory in bytes that a script is allowed to allocate. This helps prevent poorly written scripts for eating up all available memory on a server. Note that to have no memory limit, set this directive to -1.

memory_limit 的值越高，即单个 PHP 进程占用的内存越多，系统能够并发处理的请求越少。例如，一个 2G 内存的机器

memory_limit 设为 128M, 则同时最多能处理 16 个请求
memory_limit 设为 256M, 则同时最多能处理 8 个请求
memory_limit 设为 512M, 则同时最多能处理 4 个请求
memory_limit 限制了每个 PHP 进程的内存占用上限，而不是为每个进程分配了固定的内存。所以，并不会因为 memory_limit 设置越大，导致并发数出现降低。

事件

  1. 生产环境的日志出现大量的内存溢出日志,并且都是同一个接口

  2. 先分析init_get(‘memory_limit’)大小,

  3. 使用memory_get_usage()记录脚本的内存使用状况,

  4. 对比内存与memory_limit的配置,确定是在哪一步变化最大

  5. 先增大ini_set(‘memory_limit’,51210141024);

  6. 发现还是内存溢出

  7. 那么这种情况就是可能脚本代码的某个位置出现数据异常

  8. 最终定位到是某个sql查询在某个条件下,查询了几百万数据

大部分溢出原因

  1. 查询了大量的数据;
  2. 死循环
  3. 声明了大量数据的变量声明后未释放（数组,对象）

优秀的php写手

  1. 无用的变量尽量使用unset销毁
  2. unset变量销毁,只有变量的引用计数为0才能释放内存
  3. 应当尽可能减少静态变量的使用，在需要数据重用时，可以考虑使用引用(&)
  4. 一个对象使用完，要及时调用析构函数（__destruct()）
  5. 数据库操作完成后，要马上关闭连接；
  6. unset()函数只能在变量值占用内存空间超过256字节时才会释放内存空间。(PHP内核的gc垃圾回收机制决定)
