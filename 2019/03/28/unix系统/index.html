unix系统

发表于 2019-03-28 | 分类于 apue

登陆

  1. 当你登陆的时候，将会访问/etc/passwd文件对比密码以及帐号名称

  2. /etc/passwd 由7个分号切割，用户名;加密的密码;用户Id;组Id;说明域;home目录;shell脚本程序

    文件与目录

  3. 操作目录函数:include dirent.h使用opendir与readdir函数操作目录

  4. opendir需要DIR结构接收返回,readdir需要dirent结构接收返回

示例

工作目录

每一个进程都有自己的工作目录,进程也可以使用chdir变更自己的工作目录

家目录

每一个用户都有属于自己的家目录,home directory

输入与输出设备

文件描述符

  1. 文件描述符是一个非负整数,是kernel用来识别文件的,当打开一个文件或创建一个文件，kernel都会返回文件描述符
  2. 基本输入,基本输出,基本错误,每个进程执行的时候都会打开分别对应数字0,1,2
  3. 无缓存i/o由open,read,write,lseek,close这些函数提供

进程id

  1. 进程id由唯一的非负整数构成，使用getpid()函数获取,return pid_t 数据类型

进程控制

  1. 进程控制函数有三类主函数:fork,exec,waitpid

  • 我们使用fork()来创建一个新的进程,新进程复制调用者,我们称调用者为父进程,新进程为子进程,fork返回非负整数的进程id,子进程返回0,一次调用,两次返回

  • waitpid是父进程用来等会子进程执行完成，返回子进程的状态&status

线程与线程ids

多线程使在多处理器上平行计算成为可能，一个进程内的所有线程共享相同的地址空间,文件描述符,堆(stacks),进程相关属性

错误处理

  • 文件头<errno.h>定义一系列的错误码，以及对应的错误信息
  • errno错误通常为负数，但不会等于0
  • 常量EACCESS通常为权限问题
  • strerror函数可以取得errno对应的错误信息,perror可以基于errno产生一个错误信息在基本错误上

错误恢复

  • 致命错误将会停止进程的运行,非致命错误可以恢复运行,但是很不稳健
  • 资源相关的非致命错误包括:EAGAIN, ENFILE, ENOBUFS, ENOLCK, ENOSPC, EWOULDBLOCK,ENOMEM,EBUSY,EINTR

用户鉴定

  • user ID 是unix系统唯一的数字id，在/etc/passwd文件中,超级用户或root的user ID等于0,使用getuid函数获取
  • group Id /etc/passwd中的group ID 数字也有在/etc/group找到,使用getgid获取
  • 补充组ids,操作系统允许user属于其他组。

信号

信号是进程在某些条件下发生问题,通知进程用的,例如,当被除数为0时,SIGFPE信号将发送给进程，进程有三种选择处理

  1. 忽略信号
  2. 让默认的动作发生,处理被除数为0时默认将会终止进程
  3. 提供函数处理

    时间

  4. 日历时间，从1970算起
  5. 进程时间，也称cpu时间，

进程时间

  1. 时钟时间
  2. 用户CPU时间
  3. 系统CPU时间

时钟时间（有时称为挂钟时间）是进程运行所需的时间，其值取决于系统上运行的其他进程的数量。每当我们报告时钟时间时，测量都是在系统上没有其他活动的情况下进行的。

用户CPU时间是归因于用户指令的CPU时间。

系统CPU时间是代表进程执行时归属于内核的CPU时间。例如，每当进程执行系统服务（例如读取或写入）时，执行该系统服务的内核所花费的时间就会向该进程收费。用户CPU时间和系统CPU时间之和通常称为CPU时间。

可以轻松测量任何进程的时钟时间，用户时间和系统时间：只需执行time（1）命令，time命令的参数就是我们要测量的命令
